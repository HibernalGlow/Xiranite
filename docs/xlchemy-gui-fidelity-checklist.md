# Xlchemy GUI 高还原迁移清单

本清单是 Xlchemy GUI 重构的完成门槛。来源以原项目 `D:/1VSCODE/Projects/ImageAll/Xlchemy/frontend/src` 为准；`ref/node3/xlchemcy` 仅提供视觉参考，不覆盖真实工程功能。相同功能优先逐组件从 Svelte 转换为 React，不重新发明交互。

## 验收规则

- [ ] 每个功能同时具备 React 实现、交互测试和“原项目参考图 + 当前节点 UI”对照证据。
- [ ] 不因节点化删除原项目功能；确实不适用的应用壳功能必须写明替代位置。
- [ ] 使用 Xiranite 已安装的 shadcn 组件，禁止手搓 Tabs、Switch、Select、Checkbox、Progress。
- [ ] collapsed / compact / portrait / regular / expanded / workspace 全部可用。
- [ ] workspace 宽屏不能压成三条窄栏；主输入区必须拥有最大可用宽度。
- [ ] 不以折叠隐藏常用配置；低频配置进入共享配置管理，而不是“更多”菜单。

## 1. 输入文件工作区（必须忠实迁移）

来源：`cards/InputFilesCard.svelte`

- [ ] 完整工具栏：添加文件、添加文件夹、清空、移除已完成、删除已选、显示/隐藏原图预览。
- [ ] 摘要 Badge：项目总数、总大小、已选数量。
- [ ] 排序字段：名称、扩展名、大小、目录。
- [ ] 升序/降序切换。
- [ ] 列表/文件树使用明确的选中态切换器。
- [ ] 列表模式使用真实 Table：批量选择、缩略图、名称、扩展名、大小、目录。
- [ ] 表头可排序并显示方向。
- [ ] 文件行多选、右键菜单和路径提示。
- [ ] 文件树模式保留公共根目录、文件夹展开、层级缩进、文件夹半选/全选、文件数量与聚合大小。
- [ ] 文件与文件夹选择会同步到转换输入，而不是仅改变视觉状态。
- [ ] 原图缩略图使用 `host.localFiles.getUrl()`，失败时退化为文件图标。
- [ ] 空态保留说明、添加文件和添加文件夹两个主动作。
- [ ] 运行时底部动作变为主动取消；空输入时禁用转换。
- [ ] compact/portrait 不删除文件管理能力，可切换为精简行或底部 Sheet。
