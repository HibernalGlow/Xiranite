# CLI Naming And Guided UI

Xiranite node packages expose independent command-line tools. The current public command naming policy is:

```ts
nodeCliName("repacku") // xrepacku
nodeCliName("lata")    // xlata
```

The policy lives in `@xiranite/cli-runtime`:

- `NODE_CLI_PREFIX`: current prefix, now `x`
- `LEGACY_NODE_CLI_PREFIX`: old compatibility prefix, now `xiranite-`
- `nodeCliName(nodeId)`: formats the public command name
- `normalizeNodeCliName(value)`: resolves `xrepacku`, `xiranite-repacku`, or `repacku` to `repacku`

## Why This Exists

Most CLI frameworks can customize display names, help output, aliases, and subcommands. They do not make `package.json#bin` dynamic.

`package.json#bin` is a static npm manifest field. It cannot reference variables like `${prefix}${name}`. Because every node package is independently installable, each package still needs a real static `bin` entry.

To avoid hand-editing 26 package manifests, Xiranite uses a small sync script:

```powershell
bun run sync:cli-bins
```

The script rewrites every `packages/nodes/<id>/package.json` bin from `nodeCliName(id)`. If the prefix later changes, update `NODE_CLI_PREFIX` and rerun the script.

## Guided UI

`lata` is the core guided command-line application. Other commands should reuse it instead of implementing their own Taskfile selector.

The current rendering layer intentionally uses maintained terminal libraries:

- `chalk` for terminal colors
- `boxen` for Rich-like bordered panels
- `string-width` for CJK-aware width measurement
- `citty` for explicit Typer-style commands

Xiranite-owned code only defines the workflow and visual convention:

- show the active Taskfile path
- render `Taskfile 任务选择器`
- list tasks with colored task names and descriptions
- prompt for a numbered choice
- execute the selected task
- render failed commands in a red error panel
- ask whether to continue selecting tasks

## Changing Command Names

To switch from `xrepacku` to another scheme:

1. Edit `NODE_CLI_PREFIX` in `packages/cli-runtime/src/index.ts`.
2. Run `bun run sync:cli-bins`.
3. Run `bun run build:packages`.
4. Reinstall shims with `bun scripts/install-cli-shims.ts`.

To remove the prefix later, set `NODE_CLI_PREFIX` to an empty string and rerun the same steps.

## Rules For Node Packages

- Use `nodeCliName("<node-id>")` for CLI `name` and `citty` `meta.name`.
- Do not hard-code `xiranite-*`, `x*`, or future command names in source files.
- Use Taskfile `vars` for commands inside guided workflows.
- Reuse `runLataTaskSelector()` for no-arg guided Taskfile flows.
- Keep `package.json#bin` generated by `bun run sync:cli-bins`.
